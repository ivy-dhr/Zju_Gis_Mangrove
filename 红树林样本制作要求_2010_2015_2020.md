# 红树林样本制作要求清单（2010 / 2015 / 2020）

> **任务目标**：针对 `48`、`49` 号格网，分别基于 `2010`、`2015`、`2020` 年真实红树林分布生成样本点。  
> **最终要求**：每个格网每年生成 `1000` 个红树林样本点，属性表至少包含 'FID'`grid_id`、`Year`、`PXLVAL` 字段。

---

## 一、任务范围

本清单仅适用于以下内容：
- `2010` 年红树林样本制作
- `2015` 年红树林样本制作
- `2020` 年红树林样本制作
- 后续为 `2025` 年 GEE 分类准备 `2020` 年样本数据

本清单**不包含**：
- `2025` 年 Sentinel-2 影像分类
- GEE 中负样本生成
- `2025` 年分类结果导出与后处理

---

## 二、数据来源要求

### 1. 必须使用的数据
- 格网数据：`RawData/fishnet_top62/fishnet_top62.shp`
- 红树林数据必须来自 Zenodo 官方 `GMW v3`：
  - `RawData/gmw_v3_2010_vec/gmw_v3_2010_vec.shp`
  - `RawData/gmw_v3_2015_vec/gmw_v3_2015_vec.shp`
  - `RawData/gmw_v3_2020_vec/gmw_v3_2020_vec.shp`

### 2. 禁止使用的数据
- 旧版或来源不明的 `gmw_v3_2020_vec.shp`
- `GMWv4东盟.shp`
- 其他非 Zenodo 官方下载版本

---

## 三、样本制作要求

### 1. 目标格网
- 只处理 `FID = 48` 和 `FID = 49`（FID=48对应`grid_id`=36，FID=49对应`grid_id`=35）
- 开始前先确认格网字段名是否确实为 `FID`
- 需从完整格网中提取出目标格网，生成 `target_grids.shp`

### 2. 年份要求
- 仅处理 `2010`、`2015`、`2020` 三个年份
- 每一年必须使用**对应年份**的红树林面数据，不能混用

### 3. 样本数量要求
- 每个格网、每个年份生成 `1000` 个样本点
- 即：
  - `2010` 年：48号格网 `1000` 点，49号格网 `1000` 点
  - `2015` 年：48号格网 `1000` 点，49号格网 `1000` 点
  - `2020` 年：48号格网 `1000` 点，49号格网 `1000` 点
- 三年合计应得到 `6000` 个正样本点

### 4. 类别要求
- 本阶段生成的全部样本均为红树林正样本
- `PIXLVAL` 字段值统一为 `1`

## 五、完整流程要求

### 步骤 1：提取目标格网
- 输入：`RawData/fishnet_top62/fishnet_top62.shp`
- 按属性筛选：`grid_id IN (48, 49)`
- 导出结果：`target_grids.shp`
- 检查导出结果中仅包含两个格网

### 步骤 2：统一坐标系
- 检查格网数据与三期红树林数据的坐标系是否一致
- 如不一致，需先统一坐标系再继续处理
- 建议统一为 `GCS_WGS_1984`

### 步骤 3：按年份裁剪红树林范围
对 `2010`、`2015`、`2020` 分别执行：
- 输入红树林面：对应年份的 `gmw_v3_XXXX_vec.shp`
- 裁剪范围：`target_grids.shp`
- 输出结果：`mangrove_XXXX_in_grids.shp`

要求：
- 输出面数据仅保留 48、49 号格网范围内的红树林
- 每一年必须单独处理，不能合并后再分年赋值

### 步骤 4：按格网生成随机样本点
对每个年份分别执行，并按格网单独控制数量：
- 在 `48` 号格网内红树林区域生成 `1000` 个点
- 在 `49` 号格网内红树林区域生成 `1000` 个点
- 再合并为当年总样本

要求说明：
- **必须保证“每个格网 1000 个点”**，不能只保证“每年总共 1000 个点”
- **不能按每个红树林小面各生成 1000 个点**，否则点数会严重超标
- 点必须落在当年红树林面范围内，不能落在格网内非红树林区域

### 步骤 5：补充属性字段
对每个年份输出样本添加以下字段：
- `Year`：短整型，值分别为 `2010`、`2015`、`2020`
- `PIXVAL`：短整型，值固定为 `1`
- `grid_id`：整数型， 'FID'编号`48` 或 `49`对应的`grid_id`（FID=48对应`grid_id`=36，FID=49对应`grid_id`=35）

要求：
- 字段名统一使用：`Year`、`PIXLVAL`、`grid_id`
- 不要写成 `year`、`pIXLVAL`、`GridID` 等其他形式

### 步骤 6：合并每年两个格网样本
每个年份分别将 `FID=48` 和 `FID=49` 号格网样本合并为一个文件：
- `samples_2010.shp`
- `samples_2015.shp`
- `samples_2020.shp`

每个文件应满足：
- 总点数约 `2000`
- 其中 `grid_id = 36` 约 `1000` 个
- 其中 `grid_id = 35` 约 `1000` 个

### 步骤 7：为 2025 年 GEE 准备 2020 样本
从 `samples_2020.shp` 中按格网拆分：
- `samples_2020_grid48.shp`
- `samples_2020_grid49.shp`

要求：
- `samples_2020_grid48.shp` 中仅包含 `grid_id = 48`
- `samples_2020_grid49.shp` 中仅包含 `grid_id = 49`
- 两个文件都必须保留 `class = 1` 字段，供后续 GEE 调用

完成后逐项检查：

- [ ] 属性表包含 `grid_id`、`Year`、`PIXLVAL`
- [ ] `PIXLVAL` 字段值全部为 `1`
- [ ] 坐标系已统一
- [ ] `samples_2020_grid48.shp` 和 `samples_2020_grid49.shp` 已正确拆分，可直接上传 GEE

## 八、关键注意事项

| 事项 | 要求 |
|------|------|
| 数据来源 | 必须使用 Zenodo 官方 `GMW v3` 数据 |
| 处理对象 | 仅处理 `48`、`49` 号格网 |
| 样本类型 | 本阶段仅生成正样本，`PIXLVAL = 1` |
| 点数控制 | 必须是“每个格网每年 1000 点”，不是“每年总共 1000 点” |
| 随机点生成 | 点必须落在对应年份红树林面内 |
| 2020 样本用途 | 需额外拆分为 GEE 上传版本 |

---

## 九、极简总结

- `2010/2015/2020` 三个年份全部使用本地矢量流程处理
- 每年分别裁剪红树林、分别生成样本、分别补字段
- 每个年份最终得到一个合并样本文件，每年约 `2000` 点
- `2020` 年样本需额外拆出 `FID=48`、`FID=49` 两个 GEE 上传文件
- 本阶段完成后，即可进入 `2025` 年 GEE 分类流程
